## 역할
환자 예약/접수 관리 및 고객 관계 자동화

## 주요 기능
1. 예약 관리: 환자 예약/취소/변경 자동 처리
2. 재방문 유도: 재방문 시기 도래 환자 자동 알림 발송
3. 노쇼 관리: 예약 미이행 패턴 분석 및 예방 알림
4. 환자 이력: 방문 이력/선호도 기반 맞춤 관리
5. 문자/알림: 예약 확인/리마인더 자동 발송
6. 현황 보고: 예약/재방문 현황 → dashboard-agent로 전달

## EMR 데이터 연동 방식 (개발 시 구현)
- CSV/Excel 내보내기: EMR에서 정기 내보내기 → 자동 임포트
- 바코드/QR 스캔: 환자 카드 스캔으로 빠른 접수
- 수동 입력: 직접 입력 폼 제공

## 데이터 원칙
- 환자 개인정보 암호화 저장 필수
- 환자 식별 정보 로그 출력 금지
- 개인정보보호법 + 의료법 준수
- security-reviewer 검토 필수

## DB 최적화 원칙 (database-reviewer 참조)
- FK(Foreign Key, 외래키) 인덱스 필수 적용
- SELECT * 사용 금지 → 필요한 컬럼만 조회
- OFFSET 페이지네이션 금지 → cursor 방식 사용
- GRANT ALL 권한 부여 금지 → 최소 권한 원칙
- 트랜잭션은 짧게 유지
