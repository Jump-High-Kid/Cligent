## 역할
환자 문진표 디지털화 및 진료 보조

## 외부 AI 연동 방식 (개발 시 구현)
- API 키 등록 방식: GPT, Gemini, Claude 등 API 키를 프로그램에 등록
- 웹 서비스 계정 연동 방식: 웹 기반 AI 서비스 계정 연결
- 하네스 역할: 등록된 AI의 출력을 의료 안전 기준에 맞게 제어 및 후처리

## 주요 기능
1. 문진표 생성: 진료 분야별 맞춤 문진표 자동 생성
2. 웹 입력: 환자가 대기 중 스마트폰으로 문진표 작성
3. 자동 요약: 작성된 문진표를 핵심 증상 중심으로 요약
4. 진료 보조: 문진 내용 기반 예상 변증(辨證) 참고 정보 제공
5. 기록 저장: 방문별 문진 기록 이력 관리

## 데이터 원칙
- 환자 개인정보 암호화 저장 필수
- 환자 식별 정보 로그 출력 금지
- healthcare-reviewer + security-reviewer 검토 필수
- AI 제공 정보는 반드시 "참고용" 명시

## 주의사항
- 문진 결과는 진단이 아닌 참고 자료임을 명시
- 개인정보보호법 준수
