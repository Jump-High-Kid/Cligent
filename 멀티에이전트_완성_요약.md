# 멀티 에이전트 완성 — 설정 요약 문서

## 1. 전체 에이전트 구조
Orchestrator (지휘자, Sonnet)
├─ medical-lead (의료 보조 총괄)
│   ├─ blog-agent (블로그/SEO)
│   ├─ inventory-agent (재고 관리)
│   ├─ schedule-agent (직원 스케줄)
│   ├─ interview-form-agent (환자 문진표)
│   ├─ clinical-advisor-agent (임상 자문, 인허가 후)
│   ├─ content-manager-agent (자료 수집/가공)
│   ├─ youtube-agent (유튜브 교육)
│   └─ crm-agent (환자 예약/CRM)
└─ advisor-lead (법률/세무 자문 총괄)
    ├─ legal-advisor-agent (의료법/노무법)
    └─ tax-advisor-agent (세무/회계)
지원 에이전트 (독립 운영)
├─ inspection-agent (로깅/모니터링/병목/할루시네이션)
├─ test-agent (배포검증/Lint/Docker/보안)
├─ token-agent (토큰 비용 추적)
├─ dashboard-agent (통합 현황판)
└─ storage-connector-agent (저장소 추상화)

## 2. 핵심 설계 원칙

### Advisor Tool
- 일반 작업: Sonnet 단독 실행 (비용 절감)
- Opus 자동 개입 조건: 임상 자문, 보안 판단, 할루시네이션 위험, 복잡한 설계

### 외부 AI 연동 (하네스)
- 모든 기능 에이전트는 사용자 등록 외부 AI 사용
- API 키 등록 방식 + 웹 서비스 계정 연동 방식 지원
- 등록된 AI 출력을 품질/의료 안전 기준으로 제어

### 지원 에이전트 독립 운영
- orchestrator 포함 전체 시스템 감시
- 특정 에이전트에 종속되지 않음
