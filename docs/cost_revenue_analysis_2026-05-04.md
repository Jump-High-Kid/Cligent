# Cligent 비용·수익 분석 (2026-05-04 baseline)

> **목적**: 1차 베타 시작 직전 현 시점 기준 비용 구조와 단계별 수익 시뮬레이션. 베타1 종료 후 (약 15일 후) cost_logs 실측 데이터로 재분석 예정.
>
> **재분석 트리거**: 1차 베타 5인 / 15일 / 누적 25편 종료 시점, 또는 cost_logs에 50편 이상 누적 시.

## 1. 베타1 기준 운영 전제

### 사용자 측
- 봉직의 + 기타 수익 합산 = **세후 약 1.1~1.2억/년** (세전 약 1.7억 추정)
- 종소세 marginal: **38% 구간** (1.5억~3억) + 지방소득세 10% = 약 42%
- 한의원 사업자 미보유 (신규 SaaS는 별도 사업자등록 필요)
- 기존 다른 사업으로 세무사 이미 의뢰 중 (SaaS 추가 시 월 10~15만원 추가)

### 가격 구조 (2026-05-04 기준)
- **베타1**: 5인 / 15일 / 누적 25편 / 제한적 무료
- **정식 출시 (베타1 종료 후)**:
  - Standard 14.9만/월 / 30건 / 1024×1024 medium / 재생성 1회 + 수정 2회
  - Pro 27.9만/월 / 80건 / 1536×1024 high / 재생성 2회 + 수정 4회
- **코호트 단순화 (2026-05-04 결정)**: ~~5인 무료 → 25인 1만원 → 50인 15%~~ → **30인까지 베타 → 바로 정식**

### 사업자 형태 권장
- **별도 일반과세자 사업자 등록** (개인)
- 간이과세자 비추 (SaaS 부가세율 4% + 매입세액공제 불가)
- 일반과세자 매입공제 가능 항목: AI API 매입 (외화 결제 제한적)·서버비·세무비
- 50인(연매출 1.13억) 도달 시 일반과세자 강제 전환되니 처음부터 일반으로 시작
- **법인 전환 시점**: 사업소득 연 3억 초과부터 (300~500인 시점)

## 2. 변동비 (편당 정확치)

### 텍스트 (Anthropic)

실측 데이터 기준 **편당 약 110~115원** (이전 추정 75원에서 갱신).

| 항목 | 토큰 | 단가 | 비용 |
|---|---|---|---|
| Sonnet 4.6 입력 (시스템 프롬프트 + 사용자 메시지) | 약 5,100 tok | $3/Mtok | 약 21원 |
| Sonnet 4.6 출력 (한국어 2,000자 본문) | 약 6,000 tok | $15/Mtok | 약 90원 |
| Haiku 4.5 메타데이터 | 약 1,500 tok | $0.80/$4 | 약 2원 |
| **합계** | | | **약 113원** |

> **핵심 학습**: 한국어 토큰 비율이 영어보다 1.5배 높음 (1글자 ≈ 1.5 token). 시스템 프롬프트 누적 (`blog.txt` + `blog_patterns.txt` + format + sasang.txt + RAG + 의료법 고지) = 약 5,100 토큰.

### 이미지 (OpenAI)

`src/pricing.py` 공식 단가 + 토큰 부가비용 (per-image와 토큰은 **별도 가산**).

**Standard (1024×1024 medium)**

| 호출 종류 | 모델 | per-image | text_input | image_output | image_input | 장당 |
|---|---|---|---|---|---|---|
| Generation | gpt-image-2 | 74원 ($0.053) | 3.5원 | 63원 ($30/Mtok × 1,500 tok) | - | **약 140원** |
| Edit | gpt-image-1.5 | 48원 ($0.034) | 1.4원 | 67원 ($32/Mtok × 1,500 tok) | 17원 ($8/Mtok × 1,500 tok) | **약 133원** |

**Pro (1536×1024 high)**

| 호출 종류 | 모델 | per-image | text_input | image_output | image_input | 장당 |
|---|---|---|---|---|---|---|
| Generation | gpt-image-2 | 231원 ($0.165) | 3.5원 | 126원 ($30/Mtok × 3,000 tok) | - | **약 360원** |
| Edit | gpt-image-1.5 | 280원 ($0.20) | 1.4원 | 134원 ($32/Mtok × 3,000 tok) | 22원 ($8/Mtok × 2,000 tok) | **약 437원** |

> **이전 분석 갈음 사항**: per-image 단가만 보고 토큰 부가비용을 무시했었음. 실제 장당 비용은 **추정의 1.6~2배**. CLAUDE.md 명시: image_input 토큰 SDK 미보고로 0 처리 중 → **현재 cost_logs는 실제보다 17~22원/장 적게 기록**. Phase 2 정확 회계 작업 우선순위 격상 권장.
>
> **gpt-image-1.5 비용 역전**:
> - Standard medium: gpt-image-2 generation $0.053 → gpt-image-1.5 edit $0.034 (35% 저렴)
> - Pro high: gpt-image-2 generation $0.165 → gpt-image-1.5 edit **$0.20** (21% 더 비쌈)
> - image_output 토큰 단가도 gpt-image-1.5가 6.7% 비쌈 ($32 vs $30)

### 편당 변동비 (평균 사용 가정)

5장 generation + 평균 1회 edit 가정. 환율 1,400원/달러.

| 항목 | Standard | Pro |
|---|---|---|
| 텍스트 (Sonnet + Haiku) | 113원 | 113원 |
| Generation × 5 | 700원 | 1,800원 |
| Edit × 1 | 133원 | 437원 |
| **편당 변동비** | **약 950원** | **약 2,350원** |
| **편당 매출** | 4,967원 | 3,488원 |
| **편당 마진** | **81%** | **33%** ⚠️ |

### Pro 풀 사용 시 적자 가능성 (핵심 리스크)

Pro 정책 = 재생성 2회 + 수정 4회 무료. 풀 사용 시:

```
Pro 풀 사용 변동비 = 1,800 (5장 generation)
                  + 360 × 2 (재생성 2회)
                  + 437 × 4 (수정 4회)
                  + 115 (본문 + 메타)
                  = 약 4,383원/편
```

**Pro 편당 매출 3,488원 < 풀 사용 변동비 4,383원 → 편당 895원 적자**.

80건 풀 사용 시 **월 약 7만원 적자/인** 가능.

> 단, 평균 사용자가 풀로 쓰지 않을 가능성 높음 (재생성 평균 0.5회 / 수정 1~2회 추정). 베타 데이터 후 정확 평균 산출 필수.

## 6. 서버 마이그레이션 로드맵

비용 대비 효율(신뢰성·안전성·편리함) 1순위 = **AWS Lightsail Seoul**.

| 단계 | 사용자 | 인프라 | 월 비용 | 작업량 |
|---|---|---|---|---|
| 베타~정식 시작 | 0~30인 | Mac mini + launchd | 0 | 0 (현 상태 유지) |
| **정식 운영** | **30~100인** | **AWS Lightsail Seoul 4GB** | **$20 (약 27,000원)** | lift-and-shift 1~2일 |
| 성장 | 100~300인 | AWS EC2 t3.medium + RDS Postgres | 15~25만 | SQLite → Postgres 마이그레이션 1주 |
| 확장 | 300~1,000인 | ECS + CloudFront + S3 | 50~80만 | 컨테이너화 2~3주 |

### Lightsail 추천 이유
1. **신뢰성**: 99.95% SLA. Mac mini 정전·인터넷 끊김 리스크 0
2. **안전성**: Seoul 리전 (PIPA 의료 데이터 국내 보관), 자동 백업 일별, SSL 자동
3. **편리함**: Ubuntu 단일 VM = Mac mini와 거의 동일 환경. lift-and-shift
4. **예측 가능**: $20 정액. 트래픽 4TB 미초과 (300인까지 안전)

### 마이그레이션 트리거
- 30인 도달 시점 OR
- Mac mini 정전·재부팅 사고 1회 발생 시 즉시

### 스토리지 분리 시점
- 100인 도달 시점 OR `data/cligent.db`가 5GB 초과 시
- 이미지 b64를 DB에서 분리 → S3로 이전
- **확인 필요**: 현재 DB에 b64 저장 중인지 / 우선순위 높음

### 대안 — Naver Cloud Server
- PIPA 의료 데이터 국내 보관 강화 시 우선
- 비슷한 가격 (월 3만원대)
- 단점: 영문 SDK 약함, 글로벌 통합성 X
- 추천 시점: 100인+ 시점에 PIPA·의료법 강화 시 이전 검토

## 7. 베타1 후 재분석 시 확인할 데이터

### 필수 측정 (cost_logs 테이블 SQL)

```sql
-- 1. kind별 평균 변동비 (편당)
SELECT kind, COUNT(*) as n, AVG(cost_usd) as avg_usd, SUM(cost_usd) as total_usd
FROM cost_logs
WHERE created_at > datetime('now', '-30 days')
GROUP BY kind;

-- 2. 사용자별 평균 사용량
SELECT clinic_id,
       COUNT(DISTINCT blog_session_id) as blogs,
       COUNT(DISTINCT image_session_id) as image_sessions,
       SUM(cost_usd) as total_cost_usd
FROM cost_logs
WHERE created_at > datetime('now', '-30 days')
GROUP BY clinic_id;

-- 3. 이미지 edit / regen 평균 횟수 (Pro 적자 위험 확인)
SELECT
  CASE WHEN kind = 'openai_image_init' THEN 'init'
       WHEN kind = 'openai_image_regen' THEN 'regen'
       WHEN kind = 'openai_image_edit' THEN 'edit'
  END as op,
  COUNT(*) / COUNT(DISTINCT blog_session_id) as avg_per_blog
FROM cost_logs
WHERE kind LIKE 'openai_image_%'
  AND created_at > datetime('now', '-30 days')
GROUP BY op;
```

### 핵심 수치 (베타1 후 갱신할 가정)

1. **편당 평균 텍스트 비용** — 현재 추정 113원, 실측치로 갱신
2. **편당 평균 이미지 비용** — 5장 generation + 평균 edit·regen 횟수 반영
3. **Pro 사용자 평균 edit 횟수** — 적자 위험 정량화 (현재 가정 1회)
4. **OpenAI 청구서 vs cost_logs 차이** — image_input 토큰 미반영 폭 확인 (Phase 2 정확 회계 우선순위 결정)
5. **사용자 비율 Standard vs Pro** — 현재 70:30 가정. 실제 데이터로 갱신
6. **사용자별 평균 사용 건수** — 현재 풀 사용 가정 (Standard 30 / Pro 80). 실제는 더 낮을 가능성

### 정책 조정 후보 (베타1 데이터 보고 결정)

1. **Pro 가격 인상 OR edit 무료 한도 축소**
   - 평균 edit ≥ 3회 → 즉시 조정 필요
   - 평균 edit ≤ 2회 → 현재 정책 유지
2. **Pro high quality 옵션 재검토**
   - gpt-image-1.5 edit이 generation보다 비싼 역전 구조
   - 1024×1536 medium으로 다운그레이드 시 비용 30% 절감 가능
3. **이미지 토큰 정확 회계 (Phase 2)**
   - 현재 image_input 토큰 0 처리 → 실제 OpenAI 청구서 대비 +5~10% 누락
   - 베타1 후 OpenAI 콘솔 청구서와 cost_logs 대조 필수
