## 오전 세션 — 설계 & 브레인스토밍

### 1. 설치형 데스크탑 앱 전환 준비 설계

웹앱을 나중에 Windows 설치형(.exe)으로 전환할 때 재공사를 최소화하기 위한 구조 설계.

**결정 사항:**
- 설치형 방식: **Electron 래퍼 + Supabase (A안)** 채택
- Windows 우선 타깃
- 데이터 공유 방식: 전자 차트 추출 / 엑셀 업로드 / 수동 입력 → Supabase 클라우드 저장 후 공유

**설계한 4단계 준비 작업:**

| 단계 | 내용 | 난이도 |
|------|------|--------|
| 1 | API `/api/` 접두사 분리 + URL 환경변수화 | 낮음 |
| 2 | Repository 패턴 도입 (SQLite → Supabase 교체 비용 제로) | 중간 |
| 3 | 역할 Enum + 엔드포인트 태그 심기 | 낮음 |
| 4 | 경로/포트/시그널 정리 (Electron 번들링 안정화) | 낮음 |

> 상세 스펙: `docs/superpowers/specs/2026-04-16-desktop-ready-design.md`

### 2. 대시보드 앱 구조 브레인스토밍

한의원 운영 전반을 지원하는 대시보드 앱의 구조를 설계했다.

**핵심 결정:**
- 일부 기능은 공통 / 일부는 원장 전용 / 일부는 직원 설정 가능한 **모듈 기반 구조** 채택
- 웹앱과 설치형 모두 동일한 구조 사용 (서버 위치만 다름)

**이벤트 처리 & 데이터 송수신 구조:**
```
브라우저 (HTML/JS)
    ↕ fetch() REST + EventSource SSE
FastAPI 서버
    ↕
데이터 소스 (config.yaml / JSON / 추후 DB)
```

| 상황 | 방식 |
|------|------|
| 통계 카드, 목록 조회 | fetch() REST GET |
| 블로그 생성 실시간 응답 | EventSource SSE (기존 유지) |
| 설정 저장, 모듈 토글 | fetch() REST POST |

**역할 기반 모듈 구조:**
- **원장(owner):** 전체 모듈 접근
- **직원(staff):** 원장이 허용한 모듈만 표시
- 모듈 기본 권한 → `config.yaml`
- 직원별 개별 설정 → `data/staff_permissions.json`

### 3. 환경 설정 이슈 해결

- **증상:** `401 Invalid authentication credentials` API 오류
- **원인:** worktree에 `.env` 파일 없음 (`.gitignore` 대상 파일은 자동 복사 안 됨)
- **해결:** 원본 프로젝트에서 워크트리로 `.env` 수동 복사
- **Instinct 등록:** "worktree 작업 시 `.env` 복사 전 사용자에게 필요 여부 확인"

### 6. 신규 API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/` | 대시보드 (기존 `/`는 블로그 생성기였으나 변경) |
| GET | `/blog` | 블로그 생성기 (기존 `/`) |
| GET | `/api/modules/my` | 내 역할에 허용된 모듈 목록 |
| GET | `/api/modules/info` | 전체 모듈 정보 (원장 설정 화면용) |
| POST | `/api/modules/config` | 직원 모듈 권한 저장 (원장 전용) |
| GET | `/api/blog/stats` | 블로그 생성 통계 (대시보드 카드용) |

### 7. 역할 기반 모듈 시스템 동작 방식

```
페이지 로드
    → GET /api/modules/my?role=owner
    → 허용 모듈 ID 목록 수신
    → [data-module] 속성 가진 카드 순회
    → 비허용 모듈 숨김
```

**테스트 결과:**
```
owner   : 10개 모듈 전체 접근
staff_001(이간호사): 7개 (매출 카드 2개, 인사이트 숨김)
staff_002(김데스크): 5개 (매출, 인사이트, 차트, 블로그 숨김)
```

**URL 파라미터로 역할 전환 (임시, 추후 로그인으로 교체):**
- `/?role=owner` — 원장 모드
- `/?role=staff&staff_id=staff_001` — 직원 모드

### 8. 블로그 생성 이력 연동

블로그 생성 완료 시 자동 저장 → 대시보드 카드에 실시간 반영.

**저장 데이터 (`data/blog_history.json`):**
```json
{
  "id": 1,
  "keyword": "소화불량 한방 치료",
  "tone": "전문적",
  "char_count": 1850,
  "cost_krw": 12,
  "created_at": "2026-04-16T21:13:29"
}
```

**대시보드 카드 표시:**
- 이번 달 생성 편수
- 누적 생성 편수
- 최근 3개 주제 목록

**구현 방식:** SSE 스트림 래퍼(`_stream_and_save`)가 `done` 이벤트를 감지하여 이력 저장 — 프론트엔드 코드 변경 없음.
