## P2 (Phase 2 항목)

### 4. BYOAI 온보딩 가이드 영상
**What**: 고객이 Claude API 키 발급 → .env 파일에 입력하는 과정 영상.
**Why**: 기술에 익숙하지 않은 한의원이 시작부터 막히면 이탈.
**Pros**: 고객 스스로 설치, 지원 부담 감소.
**Cons**: 영상 제작 시간 30분-1시간.
**Context**: MVP 베타 테스트 시 원장님 본인이라 불필요. SaaS 고객 1호 이전에 필수.
**Depends on**: 베타 테스트 완료 후.

---

### 5. 휴머나이저 (AI 냄새 제거)
**What**: 생성된 블로그 글을 더 자연스러운 사람 문체로 변환.
**Why**: AI 생성 글 특유의 반복 패턴, 단조로운 리듬 제거.
**Pros**: 블로그 품질 향상, 독자 신뢰도.
**Cons**: 추가 API 호출 = 비용 증가.
**Context**: 베타 테스트 10편 결과로 필요성 판단. 글 샘플 10개+ 후 활성화.
**Depends on**: 베타 테스트 완료.

---

### 6. JWT 인증 + API 키 암호화
**What**: 멀티유저 전환 시 필요한 인증 시스템.
**Why**: 현재 로컬 전용이라 불필요. SaaS 공개 시 필수.
**Pros**: 다중 사용자 지원.
**Cons**: 구현 복잡도 높음 (M 에포트).
**Context**: MVP는 로컬 전용, 인증 없음. 멀티유저 전환 시 반드시 추가.
**Depends on**: 고객 수 증가 후.

---

### 7. 대시보드 중앙 현황판
**What**: 여러 기능(블로그, CRM, 재고 등)을 한 화면에서 확인.
**Why**: 기능 2개+ 생기면 개별 화면보다 대시보드가 효율적.
**Context**: 블로그 생성기 단독으로는 불필요.
**Depends on**: 2번째 기능(CRM 또는 재고) 완성 후.

### 8. 10개 AI 모델 어댑터 (GPT, Gemini 등)
**What**: Claude 외 다른 AI 모델도 선택 가능하게.

**Context**: 커스터머 요청 기반으로 추가. MVP에 불필요.
